# GridSwifts UI Guidelines

## Visual Identity

GridSwifts should look like premium sports software.

Primary style:

- dark interface
- orange highlights
- white text
- black backgrounds
- subtle gradients
- modern cards
- field-inspired visuals
- team colors
- logos
- strong sports typography

## Inspiration

- ESPN
- NFL Game Center
- Hudl
- Apple
- Linear
- Notion
- modern broadcast graphics

## Design Quality Bar

Every screen should be screenshot-worthy.

Avoid boring developer UI.

Avoid:

- huge unstyled forms
- plain grey tables
- cramped layouts
- inconsistent spacing
- default browser controls where custom controls are better
- unclear hierarchy

Prefer:

- cards
- tabs
- timeline elements
- field diagrams
- stat cards
- split panels
- keyboard-friendly forms
- smooth hover states
- clear empty states
- polished loading states

## Live Scoring UX

Live scoring is time-critical.

The operator must never be overwhelmed.

Rules:

- show only relevant fields
- auto-focus the next logical input
- Enter advances
- Ctrl+Enter saves
- keyboard shortcuts for play types
- warnings are visible but not intrusive
- errors explain exactly what is wrong

## Smart Play Wizard

The play editor should behave like a guided assistant.

Example:

```text
Play Type
  ↓
Pass
  ↓
Complete / Incomplete / Sack / Interception
  ↓
Only fields for the chosen result
```

## Dashboard

The dashboard should feel like a professional operations center.

It should show:

- active games
- recent games
- team stats
- quick actions
- database status
- build/version info
- quality warnings

## Broadcast Views

Broadcast screens must be usable in livestreams.

They should have:

- clean scorebug
- large readable text
- team logos
- possession indicator
- field visualization
- play animation
- sponsor areas
- smooth transitions

## Tables

Tables should be clean and readable.

Use:

- sticky headers where useful
- zebra rows subtly
- compact stat columns
- team/player identity cards
- sorting
- filtering

## Accessibility

- sufficient contrast
- keyboard support
- large touch targets
- readable font sizes
- no critical information by color alone
